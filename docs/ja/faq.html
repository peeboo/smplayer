  1. SMPlayer とは何ですか? また MPlayer とは何ですか?

    両者はまったく別のアプリケーションです。MPlayer は linux 用に開発されたマルチメディア プレーヤーで、他の OS にも移行されています。ほぼ何でも再生できる上、たくさんのフィルタがあるなど、たいへん便利です。しかし、一般のユーザーにとって、利用するのは困難です。

    幸いなことに、MPlayer は別のアプリケーションと一緒に動作させることができます (ビデオ ウィンドウを他のアプリケーションの GUI に埋め込むと、そのアプリケーションが送信するコマンドに従います)。そこで SMPlayer は、ビデオの再生に MPlayer を使用しているのです。

    ご覧になるウィンドウ、メニュー、環境設定ダイアログなどの部分が、SMPlayer です (ただし MPlayer を制御する機能など、ご覧にならない部分もあります)。しかし SMPlayer は、再生されるファイルについて何も、コーデックが何なのかすら知りません。一方 MPlayer は、ファイルを開き、デコードし、ビデオを表示し、オーディオを再生します。また、字幕も MPlayer によって読み取り・構文解析・表示が行われます。

  2. DVD メニューはどこですか?

    現在、MPlayer は DVD メニューに対応していません…。dvdnav サポートと一緒にコンパイルされていれば可能ですが、私が以前試したときはうまく動作しませんでした。DVD メニューはありませんが、タイトル、チャプタ、オーディオ トラック、字幕などは SMPlayer のメニューから選択できます。

  3. 出力ドライバ (directx、xv、x11、gl、alsa、oss など) とは何ですか? どれがよいですか?

    MPlayer には、ビデオ・サウンド用の出力ドライバが複数用意されています。SMPlayer の [環境設定 -> 全般] で使用するドライバを選択できます。

    linux なら xv、windows なら directx が高速かつ安定していておすすめです。ただし、Windows Vista で directx を使用すると、Aero が無効になってしまうのでご注意ください。

    x11 や directx:noaccel は不安定でおすすめできません。特に directx:noaccel を全画面モードで使用すると最悪です。

    gl と gl2 はグラフィックカードの 3D アクセラレーションを使用します。directx:noaccel や x11 よりは安定していますが、directx や xv ほどではありません。Windows Vista をお使いの方には gl/gl2 がおすすめです。

    サウンドに関しては、linux では通常 oss か alsa を使用します。esd や arts は CPU を浪費するのでおすすめできません。

    windows 用のサウンド ドライバには dsound と win32 があります。dsound はビデオとオーディオの同期化に支障が生じることがあるそうなので、dsound で何か問題があれば win32 を試してみてください。

    他のドライバは、ビデオをイメージ ファイルとして保存したり、サウンドをディスクに書き込んだりします。これらのドライバは SMPlayer のサポートの対象外ですので、納得した上でお使いください。

  4. NVidia 製カードと gl/gl2 が動作しません、なぜですか?

    NVidia 製カードと gl/gl2 は、Windows での動作に問題があるようです。MPlayer のビデオ ウィンドウが別のアプリケーションに埋め込まれていると、gl/gl2 が動作しません。残念ながら、gl/gl2 の代わりに directx を使用するか、[環境設定 -> 詳細設定] にある MPlayer のウィンドウを使用するオプションをチェックするほかに打つ手はありません。

    追記: この問題は MPlayer r25972 で修正されています。

  5. スクリーンショット オプションが無効です、なぜですか?

    [環境設定 -> 全般] でスクリーンショットを格納するフォルダを選択すると、スクリーンショット オプションは有効になります。

  6. ビデオを停止してからでないとオーディオ トラックが変更できません。なぜですか?

    mp4 や ogm などの一部のフォーマットでは、MPlayer の起動時にオーディオ トラックを変更することができません。オプション "オーディオ トラックの高速切り替え" をチェック解除すれば、オーディオを変更できるようになります。

  7. キー ショートカットを変更するには?

    [環境設定 -> キーボードとマウス] でキー ショートカットを変更できます。

  8. Qt とは何ですか?

    Qt は、クロスプラットフォームでの開発に対応した C++ ライブラリで、SMPlayer の開発に使用されています。Qt なら、Windows でも、Linux でも、Mac OS でも、同じソース コードのままでコンパイルできます。詳細: http://trolltech.com/products/qt

  9. ビデオ フォーマットによってはオーディオだけでビデオがありません。なぜですか? コーデックが必要ですか?

    MPlayer は、「届いたその日から」ポピュラーなフォーマットを再生できます。しかし、未対応のコーデックを再生するには、バイナリ コーデックのパッケージのインストールする必要があります。パッケージは MPlayer のダウンロード ページにあります。

    Windows をお使いの方は、コーデックを mplayer/codecs サブディレクトリにインストールしてください。linux ディストリビューションをお使いの方には、w32codecs などといった名前のパッケージが用意されています。

  10. [開く -> 閉じる] と [開く -> 終了] の違いは何ですか?

    メニュー [オプション] のオプション "システム トレイにアイコンを表示" がチェックされていなければ、違いはありません。どちらもアプリケーションを終了します。

    オプションが有効なら、[開く -> 閉じる] は現在のファイルを停止して、SMPlayer を起動させたままアプリケーションを非表示にします。[開く -> 終了] はシステム トレイ アイコンの有無にかかわらず、本当にアプリケーションを終了します。

  11. オーディオ CD を再生するには?

    オーディオ CD には version 0.5.18 から対応しています。

    ただし、Windows ではまだ再生できません。

  12. ブックマークはどこですか?

    まだ実装されていません。

  13. ファイルを再生しようとしても何も起こりません。なぜですか?

    MPlayer がファイルを再生できていないものと思われます。mplayer のログ ([オプション -> ログの表示 -> MPlayer]) を開けば、最後の行にエラー メッセージが見つかるかもしれません。

    Windows で一部の MPlayer ビルドを使用すると、しばしばそうなります。これはその MPlayer ビルドが対応していない機能を SMPlayer が使用しようとするためです。エラー メッセージは、解決への糸口となります。キャッシュに関してうまくいかない場合は、[環境設定 -> パフォーマンス] でキャッシュを無効にしてみてください。fontconfig に関してうまくいかない場合は、システム フォントの代わりに ttf フォントを使用してみてください。

  14. ビデオ イコライザはあるのにオーディオ イコライザがないのはなぜですか?

    MPlayer には、起動時に明るさやコントラストなどを変更するスレーブ コマンドがあるため、ビデオ イコライザは簡単です。しかし、オーディオ用のコマンドはありません。

  15. TV に対応する予定はありますか?

    よい考えですが、まだ対応する予定はありません。現在私は TV カードを持っていないので、実装できないのです。

  16. メニュー オプションを使用すると、ビデオが一瞬真っ暗になることがあります。なぜですか?

    オプションの中には、MPlayer のプロセスを一度停止させてからパラメータを追加して再び起動しなければならないものがあります。そのため、再生が一時遮断されます。

  17. SMPlayer がクラッシュしてもオーディオは止まりません、なぜですか?

    SMPlayer がクラッシュしても MPlayer は起動し続けます。タスク マネージャから MPlayer のプロセスを終了させてください。

    いずれにせよ、SMPlayer がクラッシュしたら、それは修正されなければならない深刻なバグです。直ちにご報告ください。

  18. プレイリストを再生しているとき、ファイルが終了しても SMPlayer は次のファイルを自動再生しません、なぜですか?

    SMPlayer は MPlayer の出力から英語のテキスト (この場合は "Exiting... (End of file)") を検出します。お使いの MPlayer が別の言語に翻訳されていると、その言語でメッセージが表示されて正しく検出できません。残念ながら、MPlayer の言語を再コンパイルせずに変更することはできません。ただし、SMPlayer の最近のバージョンでは検出するメッセージを変更することができます。[環境設定 -> 詳細設定 -> MPlayer の言語] をご覧ください。

  19. おすすめの MPlayer のバージョンは?

    SMPlayer は MPlayer 1.0rc1 で動作します (これより前のリリースはテストしていません)。しかしこれは非常に古く、avi ファイルでのオーディオの変更や flv、ogm ファイルのシークができないなど、まったくおすすめできません。これより後に、たくさんの変更が加わっています。

    最低でも MPlayer 1.0rc2 にすべきでしょうが、この後もさらなる変更が加わっていますので、MPlayer SVN r25843 (2008-01-23) をおすすめします。

    svn の tar.bz2 はMPlayer のダウンロード ページにあります。linux なら、コンパイルはとても簡単です。

  20. 字幕をイメージではなく黒枠の上に表示させたいのですが、可能ですか?

    [ビデオ フィルタ -> 黒枠の追加]

  21. すべてのビデオで字幕を黒枠の上に表示させたいのですが、可能ですか?

    現在、[環境設定 -> 詳細設定 -> MPlayer のオプション] で expand=:::::4/3 などのビデオ フィルタを追加する以外に方法はありません。

  22. (Windows) ビデオはファースト モニタにのみ表示され、セカンドには表示されません、なぜですか?

    [directx] をビデオ ドライバ ([環境設定 -> 全般]) として使用すると、ビデオは 1 つのモニタにのみ表示されます。代わりに [directx:noaccel]、[gl]、[gl2] を使用すれば、ビデオはすべてのモニタに表示されます。

    また、[directx] を使用したままでも、[環境設定 -> 詳細設定 -> MPlayer のオプション] の [オプション] 欄に [-adapter] オプションを追加すれば可能です。-adapter 1 ならビデオがファースト モニタに表示され、-adapter 2 ならセカンド モニタに表示されるといった具合です (多分)。

  23. ビデオの開始時の音量が大きすぎます、これは修正できますか?

    [環境設定 -> 全般 -> オーディオ] のオプション [再生の直前に音量を変更する] をチェックしてください。

    このオプションには、新しいオプション ([-volume]) を追加するパッチが適用された MPlayer ビルドが必要です。Windows 用パッケージに含まれているビルドはパッチが適用されています。パッチはこちらにあります。

  24. (Windows) Windows Vista を使用していますが、ビデオを再生するたびに Aero が無効になります、なぜですか?

    これは [directx] をビデオ ドライバ ([環境設定 -> 全般]) として使用していると起こります。[gl] か [gl2] に変更するのをおすすめします (nvidia 製カードをお使いの方はこちらをお読みください)。

  25. 構成ファイルはどこですか?

    SMPlayer は smplayer.ini と smplayer_files.ini の 2 つの ini ファイルを使用します。既定では、両者は [$HOME/.smplayer] (Linux) か [C:\Documents and Settings\ユーザー名\.smplayer] (Windows XP) に格納されます。

    SMPlayer のログ ([オプション -> ログの表示]) でも確認できます。以下のような行を探してみてください。 

     * ini path: '/home/ricardo/.smplayer'
    [smplayer.ini] は SMPlayer の構成を格納し、[smplayer_files.ini] は再生したファイルの設定を格納します。
