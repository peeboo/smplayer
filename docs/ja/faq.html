  1. SMPlayer とは何ですか? また MPlayer とは何ですか?

    両者はまったく別のアプリケーションです。MPlayer は linux 用に開発されたマルチメディア プレーヤーで、他の OS にも移植されています。ほぼ何でも再生できる上、たくさんのフィルターがあるなど、たいへん便利です。しかし、一般のユーザーにとって、利用するのは困難です。

    幸いなことに、MPlayer は別のアプリケーションと一緒に動作させることができます (ビデオ ウィンドウを他のアプリケーションの GUI に埋め込むと、そのアプリケーションが送信するコマンドに従います)。そこで SMPlayer は、ビデオの再生に MPlayer を使用しているのです。

    ご覧になるウィンドウ、メニュー、環境設定ダイアログなどの部分が、SMPlayer です (ただし MPlayer を制御する機能など、ご覧にならない部分もあります)。しかし SMPlayer は、再生されるファイルについて何も、コーデックが何なのかすら知りません。一方 MPlayer は、ファイルを開き、デコードし、ビデオを表示し、オーディオを再生します。また、字幕も MPlayer によって読み取り/構文解析/表示が行われます。

  2. DVD メニューはどこですか?

    現在、SMPlayer による DVD メニューへの対応は開発中です。

  3. 出力ドライバー (directx、xv、x11、gl、alsa、oss など) とは何ですか? どれがよいですか?

    MPlayer には、ビデオ/サウンド用の出力ドライバーが複数用意されています。SMPlayer の [環境設定 -> 全般 -> ビデオ/オーディオ] で使用するドライバーを選択できます。

    linux なら xv、windows なら directx がハードウェア アクセラレーションを使用し高性能でおすすめです。ただし、Windows Vista で directx を使用すると、Aero が無効になってしまうのでご注意ください。

    x11 や directx:noaccel にはハードウェア アクセラレーションがなく、性能が悪いのでおすすめできません。特に directx:noaccel を全画面モードで使用すると最悪です。

    gl と gl2 はグラフィックカードの 3D アクセラレーションを使用します。directx:noaccel や x11 よりは高性能ですが、directx や xv ほどではありません。Windows Vista をお使いの方には gl/gl2 がおすすめです。

    サウンドに関しては、linux では通常 oss か alsa を使用します。esd や arts は CPU を浪費するのでおすすめできません。

    windows 用のサウンド ドライバーには dsound と win32 があります。dsound はビデオとオーディオの同期化に支障が生じることがあるそうなので、dsound で何か問題があれば win32 を試してみてください。

    他のドライバーは、ビデオをイメージ ファイルとして保存したり、サウンドをディスクに書き込んだりします。これらのドライバーは SMPlayer のサポートの対象外ですので、納得した上でお使いください。

  4. スクリーンショット オプションが無効です、なぜですか?

    [環境設定 -> 全般] でスクリーンショットを格納するフォルダーを選択すると、スクリーンショット オプションは有効になります。

  5. ビデオを停止してからでないとオーディオ トラックが変更できません。なぜですか?

    mp4 や ogm などの一部のフォーマットでは、MPlayer の実行中にオーディオ トラックを変更することができません。オプション "オーディオ トラックの高速切り替え" ([環境設定 -> パフォーマンス]) をチェック解除すれば、オーディオを変更できるようになります。

  6. キー ショートカットを変更するには?

    [環境設定 -> キーボードとマウス] でキー ショートカットを変更できます。

  7. Qt とは何ですか?

    Qt は、クロスプラットフォームでの開発に対応した C++ ライブラリで、SMPlayer の開発に使用されています。Qt なら、Windows でも、Linux でも、Mac OS でも、同じソース コードのままでコンパイルできます。詳細: http://trolltech.com/products/qt

  8. ビデオ フォーマットによってはオーディオだけでビデオがありません。なぜですか? コーデックが必要ですか?

    MPlayer は、「インストールしたその日から」ポピュラーなフォーマットを再生できます。しかし、未対応のコーデックを再生するには、バイナリ コーデックのパッケージのインストールする必要があります。パッケージは MPlayer のダウンロード ページにあります。

    Windows をお使いの方は、コーデックを mplayer/codecs サブディレクトリにインストールしてください。linux ディストリビューションをお使いの方には、w32codecs などといった名前のパッケージが用意されています。

  9. [開く -> 閉じる] と [開く -> 終了] の違いは何ですか?

    メニュー [オプション] のオプション "システム トレイにアイコンを表示" がチェックされていなければ、違いはありません。どちらもアプリケーションを終了します。

    オプションが有効なら、[開く -> 閉じる] は現在のファイルを停止して、SMPlayer を実行させたままアプリケーションを非表示にします。[開く -> 終了] はシステム トレイ アイコンの有無にかかわらず、本当にアプリケーションを終了します。

  10. オーディオ CD を再生するには?

    オーディオ CD には version 0.5.18 から対応しています。

    ただし、Windows ではまだ再生できません。

  11. ブックマークはどこですか?

    まだ実装されていません。

  12. ファイルを再生しようとしても何も起こりません。なぜですか?

    MPlayer がファイルを再生できていないものと思われます。mplayer のログ ([オプション -> ログの表示 -> MPlayer]) を開けば、最後の行にエラー メッセージが見つかるかもしれません。

    Windows で一部の MPlayer ビルドを使用すると、しばしばそうなります。これはその MPlayer ビルドが対応していない機能を SMPlayer が使用しようとするためです。エラー メッセージは、解決への糸口となります。キャッシュに関してうまくいかない場合は、[環境設定 -> パフォーマンス] でキャッシュを無効にしてみてください。fontconfig に関してうまくいかない場合は、システム フォントの代わりに ttf フォントを使用してみてください。

  13. TV に対応する予定はありますか?

    よい考えですが、まだ対応する予定はありません。現在私は TV カードを持っていないので、実装できないのです。

  14. メニュー オプションを使用すると、ビデオが一瞬真っ暗になることがあります。なぜですか?

    オプションの中には、MPlayer のプロセスを一度停止させてからパラメーターを追加して再び起動しなければならないものがあります。そのため、再生が一時遮断されます。

  15. SMPlayer がクラッシュしてもオーディオは止まりません、なぜですか?

    SMPlayer がクラッシュしても MPlayer は実行し続けます。タスク マネージャから MPlayer のプロセスを終了させてください。

    いずれにせよ、SMPlayer がクラッシュしたら、それは修正されなければならない深刻なバグです。直ちにご報告ください。

  16. プレイリストを再生しているとき、ファイルが終了しても SMPlayer は次のファイルを自動再生しません、なぜですか?

    SMPlayer 0.6.5 以上、 MPlayer SVN r28066 以上をご使用ください。

  17. おすすめの MPlayer のバージョンは?

    少なくとも MPlayer SVN r25843 (2008-01-23) 以降の SVN の最新のバージョンがおすすめです。

    SMPlayer は古いバージョンでもそれに応じて動作しますが、一部のオプションが動作しなくなります。

  18. 字幕をイメージではなく黒枠の上に表示させたいのですが、可能ですか?

    [ビデオ フィルター -> 黒枠の追加]

  19. すべてのビデオで字幕を黒枠の上に表示させたいのですが、可能ですか?

    [環境設定 -> 詳細設定 -> MPlayer のオプション] で expand=:::::4/3 のようなビデオ フィルターを追加してください。

    その他の方法:

    • [環境設定 -> 全般 -> ビデオ] の 全画面表示時に黒枠を追加する をチェックする
    • [環境設定 -> 詳細設定] の動作の一覧欄に add_letterbox true を追加する
  20. (Windows) ビデオはプライマリ モニターにのみ表示され、セカンダリには表示されません、なぜですか?

    [directx] をビデオ ドライバー ([環境設定 -> 全般]) として使用すると、ビデオは 1 つのモニターにのみ表示されます。代わりに [directx:noaccel]、[gl]、[gl2] を使用すれば、ビデオはすべてのモニターに表示されます。

    また、[directx] を使用したままでも、メニュー [ビデオ -> 画面] からビデオを表示する画面を選択できます。

  21. スクリーンセーバーがオフになりません、なぜですか?

    MPlayer の最近のバージョンをお使いの方は、~/.mplayer/config に以下のような行を追加してください:

    (gnome) 

    heartbeat-cmd="gnome-screensaver-command -p &>/dev/null"

    (kde) 

    heartbeat-cmd="dcop kdesktop KScreensaverIface enable false &>/dev/null && dcop kdesktop KScreensaverIface enable true &>/dev/null"

    詳細は MPlayer の manpage をご覧ください。

  22. ビデオの開始時の音量が大きすぎます、これは修正できますか?

    [環境設定 -> 全般 -> オーディオ] のオプション [再生の直前に音量を変更する] をチェックしてください。

    このオプションには MPlayer SVN r27872 以降が必要です。

  23. (Windows) Windows Vista を使用していますが、ビデオを再生するたびに Aero が無効になります、なぜですか?

    これは [directx] をビデオ ドライバー ([環境設定 -> 全般 -> ビデオ]) として使用していると起こります。[gl]、[gl2] または [direct3d] に変更するのをおすすめします。

  24. 構成ファイルはどこですか?

    SMPlayer は smplayer.ini と smplayer_files.ini の 2 つの ini ファイルを使用します。既定では、両者は [$HOME/.config/smplayer] (Linux) か [C:\Documents and Settings\ユーザー名\.smplayer] (Windows XP) に格納されます。

    SMPlayer のログ ([オプション -> ログの表示]) でも確認できます。以下のような行を探してみてください。 

     * ini path: '/home/ricardo/.config/smplayer'
    [smplayer.ini] は SMPlayer の構成を格納し、[smplayer_files.ini] は再生したファイルの設定を格納します。
